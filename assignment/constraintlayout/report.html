package com.example.constraintuiapp

import android.os.Bundle
import android.widget.Button
import android.widget.Toast
import androidx.activity.enableEdgeToEdge
import androidx.appcompat.app.AppCompatActivity
import androidx.core.view.ViewCompat
import androidx.core.view.WindowInsetsCompat

class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableEdgeToEdge()
        setContentView(R.layout.activity_main)

        ViewCompat.setOnApplyWindowInsetsListener(findViewById(R.id.main)) { view, insets ->
            val systemBars = insets.getInsets(WindowInsetsCompat.Type.systemBars())
            view.setPadding(systemBars.left, systemBars.top, systemBars.right, systemBars.bottom)
            insets
        }

        findViewById<Button>(R.id.btnBook).setOnClickListener {
            Toast.makeText(this, "Seat booking opened", Toast.LENGTH_SHORT).show()
        }

        findViewById<Button>(R.id.btnTrailer).setOnClickListener {
            Toast.makeText(this, "Trailer preview selected", Toast.LENGTH_SHORT).show()
        }
    }
}

About ConstraintLayout

ConstraintLayout is a modern Android layout manager that helps developers design flexible and responsive user interfaces with fewer nested views. Instead of placing one layout inside another many times, ConstraintLayout allows each UI component to be positioned by defining relationships, or constraints, with other components or with the parent layout. For example, a button can be attached below a text view, aligned to the start of the screen, or centered between two sides. This approach makes the layout more organized and improves performance because the view hierarchy stays flatter.

One of the main advantages of ConstraintLayout is its ability to create complex screens in a clean and efficient way. A developer can align views vertically and horizontally, set margins, control size using constraints, and even use guidelines, chains, barriers, and bias values to fine-tune the design. This gives more control when compared with simple linear or relative layouts. It is especially useful when building screens that need to look good on many different Android devices with different screen sizes and resolutions.

Another important benefit of ConstraintLayout is that it works very well with the Android Studio Design Editor. Developers can drag and drop buttons, text views, images, and other widgets onto the screen, then connect them visually using constraints. This makes interface design faster and easier to understand, especially for beginners. The generated XML code is also easy to read once the constraint rules are understood.

In summary, ConstraintLayout is a powerful, efficient, and practical layout system for Android application development. It helps create attractive user interfaces, reduces unnecessary nesting, supports responsive design, and offers excellent tool support inside Android Studio. Because of these strengths, it is widely used for modern Android UI design.

About RelativeLayout

RelativeLayout is an Android layout manager that places views according to their relationship with the parent container or with other views already inside the screen. Instead of stacking everything in one row or one column, a developer can say that one view should appear below another, to the right of another, aligned to the bottom of the screen, or centered inside the parent. This makes RelativeLayout useful when a design has elements that depend on each other's positions.

In this sample interface, the movie poster is aligned to the right side of the screen while the title, show information, and seat details are placed to its left. The text section is arranged by using rules such as below and toStartOf, and the action buttons are attached to the bottom of the parent layout. This clearly demonstrates the core idea of RelativeLayout, where each component finds its place by referencing another component or the parent container.

RelativeLayout is easy to understand and was widely used before ConstraintLayout became the preferred choice for modern Android design. It is still helpful for small and medium-sized layouts because the XML remains readable and the positioning rules are simple. However, for very complex screens, ConstraintLayout is usually more efficient and flexible. Even so, RelativeLayout remains a valuable layout for learning how view relationships work in Android UI design.

About LinearLayout

LinearLayout is one of the simplest Android layout managers because it arranges views in a single direction. By setting the orientation to vertical or horizontal, a developer can place components in a clear sequence either from top to bottom or from left to right. This makes LinearLayout very useful when the interface naturally follows an ordered flow, such as forms, menus, lists of actions, or rows of buttons.

In this sample screen, the content is built using several vertical and horizontal LinearLayout containers. The top row displays movie formats, the middle section stacks showtime rows one under another, and the bottom section places two action buttons side by side. Weight values are used to divide horizontal space evenly, which is a common LinearLayout technique. The result is a clean structure that clearly shows how rows and columns are created by nesting layouts.

LinearLayout is ideal for beginner-friendly Android projects because the XML is easy to read and the behavior is predictable. However, if many LinearLayouts are nested inside each other, the view hierarchy can become deep and harder to manage. For simple screens, it remains an excellent choice, but for larger and more flexible designs, developers often move to ConstraintLayout. Even then, understanding LinearLayout is important because it teaches the basic logic of arranging UI elements in Android.

MainActivity.kt

package com.example.frameuiapp

import android.os.Bundle
import android.view.View
import android.widget.Button
import android.widget.Toast
import androidx.activity.enableEdgeToEdge
import androidx.appcompat.app.AppCompatActivity
import androidx.core.view.ViewCompat
import androidx.core.view.WindowInsetsCompat

class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableEdgeToEdge()
        setContentView(R.layout.activity_main)

        ViewCompat.setOnApplyWindowInsetsListener(findViewById(R.id.main)) { view, insets ->
            val systemBars = insets.getInsets(WindowInsetsCompat.Type.systemBars())
            view.setPadding(systemBars.left, systemBars.top, systemBars.right, systemBars.bottom)
            insets
        }

        findViewById<View>(R.id.btnPlay).setOnClickListener {
            Toast.makeText(this, "Trailer is playing", Toast.LENGTH_SHORT).show()
        }

        findViewById<Button>(R.id.btnDetails).setOnClickListener {
            Toast.makeText(this, "Opening movie details", Toast.LENGTH_SHORT).show()
        }
    }
}

About FrameLayout

FrameLayout is an Android layout manager designed for stacking views on top of each other inside a single area. Every child view shares the same parent space, and the order in which the views are declared determines which one appears in front. Because of this behavior, FrameLayout is perfect for overlays, floating labels, media previews, fragments, loading screens, and simple layered designs.

In this sample UI, the poster background is placed first, then a semi-transparent dark overlay is added, followed by a top badge, a centered play icon, and a details panel at the bottom. All of these components exist inside the same FrameLayout container, so the screen demonstrates the main feature of FrameLayout very clearly. Instead of arranging views side by side, the layout builds depth by placing one view above another.

FrameLayout is simple, lightweight, and very useful when the design focuses on a single content area with multiple layers. It is not the best option for large screens that need complex positioning in many directions, but it works extremely well for stacked content. Learning FrameLayout helps developers understand the visual order of views and how Android can create rich interfaces by combining layers in one container.
